# kartg 🖨️

**Система учета жизненного цикла картриджей для лазерных принтеров.**

Легкое, высокопроизводительное веб-приложение для регистрации выдачи, заправки и утилизации картриджей. Предназначено для малых офисов и сервисных центров (до 10 пользователей, база до 10 000 единиц).

## ✨ Особенности

*   **Полный цикл учета:** Регистрация новых картриджей, выдача на заправку, прием после заправки, списание в утилизацию.
*   **Автоматический подсчет:** Система автоматически ведет историю заправок для каждого уникального ID.
*   **Аналитика и Отчеты:** Генерация отчетов в форматах **CSV** и **TXT** непосредственно на бэкенде (статистика за период, история конкретного картриджа, ресурс до утилизации).
*   **Архитектура:** Микросервисный подход в монолитной упаковке. gRPC внутри, REST/JSON снаружи для удобного веба.
*   **Производительность:** Написано на Go, использует SQLite в режиме WAL для высокой конкурентности при записи.
*   **Простота деплоя:** Работает в одном Docker-контейнере. База данных — единый файл.
*   **Безопасность:** JWT аутентификация, защита роутов, хеширование паролей (bcrypt).
*   **Тестирование:** Unit-тесты (81.8% models, 47.7% service), E2E тесты полного цикла.

## 🛠 Технологический стек

### Бэкенд
*   **Язык:** Go (Golang)
*   **API:** gRPC + [gRPC-Gateway](https://github.com/grpc-ecosystem/grpc-gateway) (гибридный режим: gRPC внутри, REST/JSON + Swagger для клиента)
*   **База данных:** SQLite 3 (`modernc.org/sqlite` — чистый Go драйвер)
*   **ORM:** GORM
*   **Логирование:** `log/slog`
*   **Конфигурация:** Переменные окружения (`os.Getenv`)

### Фронтенд
*   **Фреймворк:** React (Vite) + TypeScript
*   **State Management:** Zustand
*   **UI Components:** Headless подход
    *   Таблицы: `TanStack Table` (сортировка, пагинация, фильтры)
    *   Формы: `React Hook Form` + валидация
    *   Стили: `Tailwind CSS`
*   **Клиент API:** Авто-генерация типов из OpenAPI/Swagger спецификации бэкенда

### Инфраструктура
*   **Контейнеризация:** Docker & Docker Compose
*   **CI/CD:** GitHub Actions

## 🏗 Архитектура данных

### Основные сущности

#### `Cartridge` (Картридж)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `ID` | String | Уникальный идентификатор (серийный номер/штрихкод) |
| `Model` | String | Модель принтера/картриджа |
| `Status` | Enum | `InUse`, `Refilling`, `Retired` |
| `TotalRefills` | Int | Счетчик успешных заправок |
| `CreatedAt` | Time | Дата первой регистрации |
| `RetiredAt` | Time | Дата списания (если утилизирован) |

#### `Transaction` (Операция)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `ID` | UUID | ID транзакции |
| `CartridgeID` | String | Ссылка на картридж |
| `Type` | Enum | `SendToRefill`, `ReceiveFromRefill`, `Retirement`, `Registration` |
| `Timestamp` | Time | Время операции |
| `Comment` | String | Комментарий оператора |

## 🔐 Авторизация

### Учетные данные по умолчанию
```
👤 Логин: admin
🔑 Пароль: (пустой, не требуется)
```

### Упрощенный вход
- Пользователь `admin` создается автоматически при первом запуске
- Пароль не требуется (можно оставить поле пустым)
- Имя пользователя запоминается в браузере для быстрого входа
- При наличии действительного токена вход выполняется автоматически

### JWT токены
- Время жизни: 24 часа
- Хранение: localStorage
- Алгоритм: HS256

### Переменные окружения
| Переменная | Значение по умолчанию | Описание |
|-----------|----------------------|----------|
| `JWT_SECRET` | `kartg-secret-key-change-in-production` | Секретный ключ для JWT |

## 🌐 Многоязычность (i18n)

### Поддерживаемые языки
- 🇷🇺 **Русский** (ru) — язык по умолчанию
- 🇬🇧 **English** (en)

### Определение языка
1. Query параметр: `?lang=en` или `?lang=ru`
2. Заголовок `Accept-Language`
3. Язык по умолчанию: русский

### Примеры запросов
```bash
# English
curl http://localhost:8080/api/v1/health?lang=en

# Русский (по умолчанию)
curl http://localhost:8080/api/v1/health
```

### Добавление нового языка
1. Создайте файл `internal/i18n/locales/{code}.json`
2. Добавьте переводы
3. Обновите `internal/i18n/i18n.go`

## 🚀 Быстрый старт

### Требования
*   Docker & Docker Compose
*   (Опционально) Go 1.21+ и Node.js 18+ для локальной разработки без Docker

### Запуск через Docker Compose

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/your-username/kartg.git
    cd kartg
    ```

2.  Запустите сервисы:
    ```bash
    docker-compose up -d --build
    ```

3.  Приложение доступно по адресу:
    *   **Веб-интерфейс:** `http://localhost:3000`
    *   **API (Swagger UI):** `http://localhost:8080/swagger`
    *   **gRPC Gateway (REST):** `http://localhost:8080/api`

> **Примечание:** Данные сохраняются в файле `data/kartg.db` в корне проекта (благодаря монтированию тома в Docker).

## 📖 API Документация

Проект использует **gRPC-Gateway**, который автоматически генерирует REST API и Swagger спецификацию.

После запуска приложения откройте `http://localhost:8080/swagger` для интерактивной документации всех доступных эндпоинтов.

### Основные группы методов:
*   `CartridgeService`: Управление реестром картриджей.
*   `OperationService`: Проведение операций (заправка, утилизация).
*   `AnalyticsService`: Получение статистики.
*   `Export`: Скачивание отчетов (`GET /api/export/...`).

## 📊 Функционал отчетности

Система позволяет выгружать данные без визуализации на клиенте. Бэкенд формирует файл потоково и отдает его браузеру.

1.  **Отчет за период:** Количество заправок за выбранные даты.
2.  **История картриджа:** Полная хронология жизни конкретного ID.
3.  **Ресурс картриджа:** Статистика по утилизированным картриджам (сколько заправок прожили).

**Форматы экспорта:**
*   `.csv` (совместим с Excel, Google Sheets)
*   `.txt` (табулированный текст)
